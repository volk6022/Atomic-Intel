# Как вносить изменения

Прод-репозиторий. Все изменения проходят через ревью владельца.

## Поток

1. **Issue.** Опиши задачу/баг отдельным issue до начала работы.
2. **Branch.** Ветка от `main` с говорящим именем:
   `feat/<коротко>`, `fix/<коротко>`, `docs/<коротко>`, `chore/<коротко>`.
3. **Commits.** Атомарные, в настоящем времени: `fix: …`, `feat: …`, `docs: …`.
4. **Pull Request.** PR в `main` со ссылкой на issue. CI должен быть зелёным.
5. **Review.** Обязательный ревью и аппрув владельца. Прямой push в `main` запрещён.

## Правила

- **Секреты** — только через `.env` (в `.gitignore`). Никогда не коммить ключи, токены,
  прокси-креды, сессии. Перед PR проверь `git diff` на утечки.
- **Прод ≠ дев.** Полный функционал живёт в приватном upstream. Сюда не возвращай
  dev-only подсистемы без явного решения (см. `CLAUDE.md` → границы прод/дев).
- **Docker** — изменения не должны ломать `docker compose build && up`.
- **Тесты/линт** — прогони локально перед PR.

## Локальный старт

```bash
cp .env.example .env
docker compose build && docker compose up -d
```
